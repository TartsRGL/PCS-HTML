# Lesson 7 – Forms & Accessibility

This folder contains the **HTML-only code** for Lesson 7 of the **PCS-HTML** series.

It focuses on building inclusive, semantically correct forms. Forms are often styled heavily, but a solid foundation ensures interactions

 and screen readers interpret them properly without relying on visual cues alone.

It demonstrates:

-**Form Basic Setup**: Establish the correct container (`<form>`) with proper action and method attributes.

-**Explicit Labels**: Connecting inputs with visible `<label>` text using `for` and `id`. Avoid using `placeholder` as a label replacement.

-**Fieldsets**: Grouping multiple related inputs (like custom radio buttons or grouped fields) using `<fieldset>` and `<legend>`.

-**Hints & Errors**: Adding descriptive hints directly tied to inputs using `aria-describedby` and structuring error messages gracefully.

-**Composition**: Assembling the puzzle into a full form representation including the submit button.

Each commit introduces **one clear HTML decision at a time** – step-by-step.
