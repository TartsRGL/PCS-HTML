# Lesson 6 – Components & Reusability

This folder contains the **HTML-only code** for Lesson 6 of the **PCS-HTML** series.

It focuses on how to write HTML that serves as a foundation for reusable UI components.
The goal is to move away from page-specific HTML structures and start thinking in "blocks"
that can be dropped anywhere in the layout without breaking

It demonstrates:

- **Component Boundaries**: Establishing clear edges between a component and the rest of the page.
- **Internal Structure**: Defining stable sub-elements (like header, body, footer) inside the component.
- **Repetition Prep**: Writing markup that doesn't rely on strict IDs but uses classes designed for repetition.
- **Variant Prep**: Structuring markup that can safely receive variant modifiers (like BEM modifiers) in CSS.
- **Assemblage**: Grouping multiple components into a list or grid context.

Each commit introduces **one clear HTML decision at a time** – step-by-step.
