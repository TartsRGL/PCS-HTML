# Lesson 4 – Flow, Order & Reading Sequence

This folder contains the **HTML-only code** for Lesson 4 of the **PCS-HTML** series.

It focuses on how content should appear in a **meaningful source order** before any CSS styling is applied.

The goal is to understand that visual clarity often begins with correct HTML flow.

It demonstrates:

- Basic page flow through `header`, `main`, and `footer`
- Expansion of `main` with additional content blocks
- Why content order matters in HTML
- Difference between adding content and arranging content
- Reading sequence from introduction to main message to supporting details
- Why closing content belongs at the end of the main flow

Each commit introduces **one clear HTML decision at a time** – step-by-step.

---

## 📌 Commits overview

```bash
feat: establish base page flow
feat: add another section to main
refactor: arrange sections in reading order
feat: add closing section at the end
chore: final source order sanity check
```
